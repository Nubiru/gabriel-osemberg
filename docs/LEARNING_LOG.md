# Learning Log — {PROJECT_NAME}

Record of lessons learned, mistakes made, and insights gained.

---

## Format Guide

Each entry follows this structure:

```markdown
## YYYY-MM-DD: {Title}

**Category**: Architecture | Process | Testing | Performance | Collaboration | Tooling
**Context**: What was happening
**Lesson**: What was learned
**Action**: What changed as a result
```

### Categories

| Category | Description |
|----------|-------------|
| Architecture | Design patterns, module structure, dependency decisions |
| Process | Workflow, communication, planning |
| Testing | Test strategy, coverage, debugging |
| Performance | Optimization, profiling, resource usage |
| Collaboration | Human-AI coordination, code review, delegation |
| Tooling | Build system, CI/CD, development environment |
